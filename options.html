Check My Links: Options

To prevent this extension from checking a link, please enter all or part of the URL in the blacklist below. Each URL must be entered on a new line.

For example: www.mydomain.com will prevent the following links from being checked: www.mydomain.com/page1.html or www.mydomain.com/page2.html

To prevent this extension from checking a link based on innerText it contains, enter it below. Each link innerText must be entered on a new line. Note: This is case-insensitive.

For example: Logout will prevent the following links from being checked: <a href="login.aspx">Logout</a> or <a href="login.aspx">login or logout</a>


HEAD is quicker, but sometimes doesn't reflect the true HTTP status of a page, use GET if you're getting weird results. GET is required to be able to parse the document for an element corresponding to what is after the hashtag by ID.

Warnings

Hashtags can link to elements by id within the DOM. Enabling will add add the check to make sure an element exists by id to correspond to the hashtag. When enabled, if the page resolves but the corresponding element is not found, the element will be marked as a warning and shown in the console.log (F12->Console tab). Request Type: GET is required

Valid Link Caching

If you don't want to constantly re-check good (green, HTTP 200) links, enable the option below to force the extension to only check links which were previously broken (red, HTTP 404 etc). Enabling 'Valid Link Caching' vastly reduces the time it will take to repeatedly check a page.

Nofollow Links

By default, Check My Links will not check links with a link rel attribute of 'nofollow', you should apply this attribute to links you don't want the extension to trigger (such as Log Out, Sign Out links etc).
